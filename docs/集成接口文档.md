# 合同审查服务 — 集成接口文档

> 版本：v1.0  
> 接口前缀：`/api/integration/v1`

---

## 1. 调用方式

审查流程只需调用一个接口：

```
POST /api/integration/v1/review
Content-Type: multipart/form-data
```

大系统的审查页面中，**模板下拉框直接写死**不需要调接口动态获取。模板 ID 是固定映射：

### 模板编号对照表

| 模板 ID | 合同名称 | 下拉框显示文字 |
|---------|------|------|
| `32` | 第三方管理服务标准合同 | 第三方管理服务标准合同-调整版V4 |
| `33` | 技术服务标准合同 | 技术服务标准合同-调整版V4 |
| `34` | 汽车服务标准合同 | 汽车服务标准合同-调整版V4 |
| `35` | 外包服务标准合同 | 外包服务标准合同-调整版V4 |
| `36` | 网络推广服务标准合同 | 网络推广服务标准合同-调整版V4 |
| `37` | 咨询服务标准合同 | 咨询服务标准合同-调整板V4 |
| `38` | 租车服务标准合同 | 租车服务标准合同-调整版V4 |

> 用户在下拉框中选择某个合同后，发请求时把对应的 `template_id` 传入即可。

### 审查类型对照

| 编号 | 类型 | 说明 |
|------|------|------|
| `1` | 段落比对 | 模板与文档逐段 Diff，输出差异段和违规位置 |
| `2` | 规则校验 | 提取文档填充值，按标注规则逐字段校验 |
| `3` | 完整审查 | 比对 + 校验一起返回 |

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `template_id` | int | 是 | 模板编号，见上方对照表 |
| `review_type` | int | 是 | `1` 比对 / `2` 校验 / `3` 完整 |
| `file` | file | 是 | 待审文档（.docx） |

**错误码：**

| 状态码 | 含义 |
|--------|------|
| 400 | 文件非 .docx / review_type 不在 1-3 |
| 404 | 模板 ID 不存在 |
| 500 | 服务端解析或比对异常 |

## 2. 模板管理（后台用）

### 2.1 获取模板列表

大系统下拉选择模板时调用。

```
GET /api/integration/v1/templates
```

**请求：** 无参数

**当前模板清单：**

| ID | 合同名称 |
|----|------|
| 32 | 第三方管理服务标准合同-调整版V4 |
| 33 | 技术服务标准合同-调整版V4 |
| 34 | 汽车服务标准合同-调整版V4 |
| 35 | 外包服务标准合同-调整版V4 |
| 36 | 网络推广服务标准合同-调整版V4 |
| 37 | 咨询服务标准合同-调整板V4 |
| 38 | 租车服务标准合同-调整版V4 |

> **注意：** 大系统调用审查接口时，`template_id` 参数使用上表中的 ID 值。新增模板后 ID 会递增，以接口实际返回为准。

**响应 200：**

```json
[
  {
    "id": 32,
    "name": "第三方管理服务标准合同-调整版V4.docx",
    "paragraph_count": 62,
    "created_at": "2026-06-24 14:30:00"
  },
  {
    "id": 33,
    "name": "技术服务标准合同-调整版V4.docx",
    "paragraph_count": 58,
    "created_at": "2026-06-24 14:30:00"
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 模板编号，审查时传给 `template_id` |
| `name` | string | 原始文件名 |
| `paragraph_count` | int | 段落总数 |
| `created_at` | string | 上传时间 |

### 2.2 上传模板

将 .docx 模板注册到审查系统。

```
POST /api/integration/v1/templates/upload
```

**请求（multipart/form-data）：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | file | 是 | 模板文件（.docx） |

**响应 200：**

```json
{
  "id": 3,
  "name": "新模板.docx",
  "paragraph_count": 28,
  "created_at": ""
}
```

**错误响应：**

| 状态码 | 说明 |
|--------|------|
| 400 | 文件不是 .docx 格式 |

### 2.3 获取模板详情

获取某个模板的完整段落内容。

```
GET /api/integration/v1/templates/{template_id}
```

**路径参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `template_id` | int | 模板编号 |

**响应 200：**

```json
{
  "id": 1,
  "name": "服务标准合同.docx",
  "paragraphs": [
    {
      "index": 0,
      "text": "第一条 委托事项",
      "underline_ranges": [],
      "is_table_cell": false
    },
    {
      "index": 1,
      "text": "甲方委托乙方提供______服务，服务期限自______起至______止。",
      "underline_ranges": [[10, 16], [24, 30], [33, 39]],
      "is_table_cell": false
    }
  ],
  "created_at": "2026-06-20 10:30:00"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `paragraphs[].index` | int | 段落索引 |
| `paragraphs[].text` | string | 段落文本 |
| `paragraphs[].underline_ranges` | int[][] | 下划线位置（字符级） |
| `paragraphs[].is_table_cell` | bool | 是否表格内段落 |

**错误响应：**

| 状态码 | 说明 |
|--------|------|
| 404 | 模板不存在 |

### 2.4 删除模板

|--------|------|

## 3. 标注管理

标注定义了模板中哪些区域是"可填充区"，以及每个可填充区的校验规则。标注由**内部标注工具**维护，大系统一般不需要调用写接口。

### 3.1 获取模板标注

```
GET /api/integration/v1/templates/{template_id}/annotations
```

**响应 200：**

```json
[
  {
    "paragraph_index": 0,
    "start_char": 5,
    "end_char": 11,
    "zone_type": "fillable",
    "rules": "{\"required\":true,\"field_name\":\"合同编号\",\"min_chars\":5,\"max_chars\":20,\"allowed_chars\":\"alphanumeric\"}"
  },
  {
    "paragraph_index": 0,
    "start_char": 0,
    "end_char": 0,
    "zone_type": "fixed",
    "rules": "{}"
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `paragraph_index` | int | 段落索引 |
| `start_char` | int | 标注起始位置 |
| `end_char` | int | 标注结束位置 |
| `zone_type` | string | `fixed` 固定文本 / `fillable` 可填充 / `variable` 可变文本 |
| `rules` | string | 校验规则 JSON 字符串（fillable 类型时有效） |

### 3.2 保存模板标注

> **注意：** 全量替换，已有标注会被删除。

```
POST /api/integration/v1/templates/{template_id}/annotations
```

**请求（application/json）：**

```json
{
  "annotations": [
    {
      "paragraph_index": 1,
      "start_char": 8,
      "end_char": 14,
      "zone_type": "fillable",
      "rules": {
        "required": true,
        "field_name": "合同金额",
        "allowed_chars": "number",
        "amount_match_field": "大写金额"
      }
    },
    {
      "paragraph_index": 1,
      "start_char": 0,
      "end_char": 8,
      "zone_type": "fixed",
      "rules": null
    }
  ]
}
```

**rules 字段说明：**

| 规则 | 类型 | 说明 |
|------|------|------|
| `required` | bool | 是否必填 |
| `min_chars` | int | 最少字符数 |
| `max_chars` | int | 最多字符数 |
| `allowed_chars` | string | 字符类型：`chinese` / `number` / `alphanumeric` / `any` / `regex` |
| `regex` | string | 当 `allowed_chars=regex` 时的正则 |
| `field_name` | string | 字段显示名称 |
| `allowed_values` | string[] | 允许的可选值列表 |
| `match_fields` | string[] | 需保持一致的字段名列表 |
| `radio_group` | string | 单选组名称（用于 checkbox 互斥） |
| `dependent_paras` | int[] | checkbox 管辖的段落索引 |
| `amount_match_field` | string | 对应中文大写金额的 field_name |
| `amount_unit` | int | 金额单位：1=元，1000=千元，10000=万元 |

**响应 200：**

```json
{"ok": true, "count": 2}
```

## 5. 统一审查（核心接口）

> 这是大系统的主要调用接口，一步完成"上传文档 + 执行审查 + 返回结果"。

```
POST /api/integration/v1/review
Content-Type: multipart/form-data
```

**表单参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `template_id` | int | 是 | 模板编号（从模板列表接口获取） |
| `review_type` | int | 是 | 审查类型：`1` 比对 / `2` 校验 / `3` 完整 |
| `file` | file | 是 | 待审文档（.docx） |

      },
      {
        "paragraph": 2,

**compare 字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `diffs` | array | 逐段 Diff 结果 |
| `diffs[].type` | string | `equal` 一致 / `insert` 文档多出 / `delete` 文档缺失 / `replace` 被替换 |
| `diffs[].template_range` | int[2] | 模板全文中的起止字符偏移 |
| `diffs[].doc_range` | int[2] | 文档全文中的起止字符偏移 |
| `diffs[].value` | string | 差异文本内容 |
| `violations` | array | 违规项（只包含 insert/delete/replace/keyword） |
| `violations[].paragraph` | int | 模板段落索引 |
| `violations[].type` | string | `insert` / `delete` / `replace` / `keyword` |
| `violations[].template_text` | string | 模板原文 |
| `violations[].actual_text` | string | 文档实际文本 |
| `template_text` | string | 模板完整文本 |
| `document_text` | string | 文档完整文本 |
| `paragraph_mapping` | object | 模板段落 → 文档段落映射，`null` 表示该段落被整段删除 |
| `inserted_paragraphs` | int[] | 文档中多出的段落索引 |
| `absorbed` | object | 跨段落吸收映射 |
| `keyword_matches` | array | 敏感词匹配结果 |

### 5.2 review_type=2（规则校验）响应

```json
{
  "document_id": 12,
  "validate": {
    "results": [
      {
        "paragraph": 1,
        "start_char": 15,
        "end_char": 23,
        "ann_start_char": 10,
        "field_name": "合同金额",
        "actual_value": "50000",
        "rule": "必填+数字+金额(元)↔「大写金额」",
        "pass": true,
        "reason": "",
        "paragraph_text": "甲方应向乙方支付50000元整。",
        "template_paragraph_text": "甲方应向乙方支付______元整。"
      },
      {
        "paragraph": 4,
        "start_char": 5,
        "end_char": 5,
        "ann_start_char": 3,
        "field_name": "乙方签章",
        "actual_value": "",
        "rule": "必填",
        "pass": false,
        "reason": "必填字段为空",
        "paragraph_text": "乙方（盖章）：",
        "template_paragraph_text": "乙方（盖章）："
      }
    ],
    "document_paragraphs": [
      {"index": 0, "text": "第一条 委托事项"},
      {"index": 1, "text": "甲方应向乙方支付50000元整。"}
    ],
    "template_paragraphs": [
      {"index": 0, "text": "第一条 委托事项"},
      {"index": 1, "text": "甲方应向乙方支付______元整。"}
    ],
    "paragraph_mapping": {"0": 0, "1": 1},
    "inserted_paragraphs": [],
    "absorbed": {}
  }
}
```

**validate.results 字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `paragraph` | int | 模板段落索引 |
| `start_char` / `end_char` | int | 该字段在文档中的字符范围 |
| `field_name` | string | 字段名称（来自标注） |
| `actual_value` | string | 从文档中实际提取的值 |
| `rule` | string | 生效的规则描述 |
| `pass` | bool | 校验是否通过 |
| `reason` | string | 未通过时的原因说明 |
| `paragraph_text` | string | 文档中所在段落的完整文本 |
| `template_paragraph_text` | string | 模板中对应段落的完整文本 |

**校验规则说明：**

结果按 `pass` 排序（失败的在前）。字段可能因以下原因不通过：

| reason 示例 | 触发条件 |
|------|------|
| 必填字段为空 | `required=true` 且文档中该字段无内容 |
| 字数不足：最少5字符 | 填写内容短于 `min_chars` |
| 字数超限：最多200字符 | 填写内容长于 `max_chars` |
| 字符类型不符：要求number | 填写内容不符合 `allowed_chars` |
| 格式不符：需匹配 ^[\d]{18}$ | `allowed_chars=regex` 时不匹配正则 |
| 值不在允许范围内 | 填写值不在 `allowed_values` 列表中 |
| 与「字段X」不一致 | 跨字段一致性校验失败 |
| 与「大写金额」不匹配 | 数字金额与中文大写金额不匹配 |
| 「选项组」未选择任何选项 | checkbox 单选组未勾选 |
| 「选项组」只能选择一个选项 | checkbox 单选组多选 |
| 该条款已勾选，需填写内容 | checkbox 已勾选但关联字段为空 |
| 该条款未勾选，不应填写内容 | checkbox 未勾选但关联字段有内容 |

## 6. 调用示例

### cURL

```bash
# 1. 获取模板列表
curl -X GET "http://<host>/api/integration/v1/templates"

# 2. 发起完整审查（比对+校验）
curl -X POST "http://<host>/api/integration/v1/review" \
  -F "template_id=1" \
  -F "review_type=3" \
  -F "file=@/path/to/待审合同.docx"

# 3. 仅段落比对
curl -X POST "http://<host>/api/integration/v1/review" \
  -F "template_id=1" \
  -F "review_type=1" \
  -F "file=@/path/to/待审合同.docx"

# 4. 仅规则校验
curl -X POST "http://<host>/api/integration/v1/review" \
  -F "template_id=1" \
  -F "review_type=2" \
  -F "file=@/path/to/待审合同.docx"
```

### Python

```python
import requests

BASE = "http://<host>/api/integration/v1"

# 获取模板列表
templates = requests.get(f"{BASE}/templates").json()
for t in templates:
    print(f"ID={t['id']}  {t['name']}")

# 发起完整审查
with open("待审合同.docx", "rb") as f:
    resp = requests.post(
        f"{BASE}/review",
        data={"template_id": 1, "review_type": 3},
        files={"file": ("合同.docx", f, "application/vnd.openxmlformats-officedocument.wordprocessingml.document")}
    )

result = resp.json()
print(f"文档ID: {result['document_id']}")
print(f"差异段数: {len(result['compare']['diffs'])}")
print(f"违规数: {len(result['compare']['violations'])}")
print(f"校验字段数: {len(result['validate']['results'])}")

# 检查校验失败的字段
for r in result["validate"]["results"]:
    if not r["pass"]:
        print(f"  ✗ {r['field_name']}: {r['reason']}")
```

## 7. 错误码汇总

| 状态码 | 触发条件 |
|--------|------|
| 400 | 文件不是 .docx / review_type 不在 1-3 范围内 |
| 404 | 模板 ID 或文档 ID 不存在 |
| 500 | 服务端解析或比对异常 |
