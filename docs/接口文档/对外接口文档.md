# 合同审查服务 — 对外接口文档

## 概述

本服务用于**自动审查合同文档**：将用户上传的 Word 合同（.docx）与预设的合同模板进行比对，自动发现以下问题：

- **内容篡改**：合同正文是否被修改、删除或新增条款
- **字段校验**：金额、日期、签章等关键填充字段是否符合规范
- **敏感词检测**：是否包含不合规词汇（如"保费收入"等）

外部系统集成后，用户选择一个合同模板，上传待审合同文件，即可获得审查结果。

---

## 集成流程

```
1. GET /templates       →  拿到可用模板列表，渲染下拉框
2. 用户在 UI 选择模板，上传 .docx 文件，选择审查类型
3. POST /review         →  提交审查，拿到审查结果
4. 前端根据结果渲染差异高亮、校验结论
```

只需 2 个接口。

---

## 接口约定

**基础路径：** `/api/integration/v1`

**鉴权：** 暂无（后续可加 API Key 头）

**统一响应格式：** 所有接口返回 JSON，结构固定为：

```json
{
  "code": 200,
  "message": "success",
  "data": <具体业务数据, 类型见各接口>,
  "timestamp": 1719312000000
}
```

- `code` — HTTP 状态码。`200` 表示成功，`4xx` / `5xx` 表示失败
- `message` — 成功时固定为 `"success"`，失败时为具体错误描述
- `data` — 成功时为业务数据（对象/数组），失败时为 `null`
- `timestamp` — 响应时刻的 Unix 毫秒时间戳

**错误响应示例：**

```json
{"code": 404, "message": "模板不存在", "data": null, "timestamp": 1719312000000}
```

**前端注意事项：**

- 判断成功/失败**不要只看 HTTP 状态码**，也看 `code` 字段
- 错误信息直接取 `message` 展示给用户即可
- 文件上传统一用 `multipart/form-data`，不是 JSON

---

## 接口 1：获取模板列表

### 用途

页面加载时调用，获取可供用户选择的合同模板列表。返回值中的 `id` 将用于调用审查接口。

### 请求

```
GET /api/integration/v1/templates
```

无需参数。

### 成功响应

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 32,
      "name": "第三方管理服务标准合同-调整版V4.docx",
      "paragraph_count": 62,
      "created_at": "2026-06-24 14:30:00"
    }
  ],
  "timestamp": 1719312000000
}
```

### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| data | array | 模板列表，按上传时间排序 |
| data[].id | int | **模板唯一标识**，调用审查接口时需要传入 |
| data[].name | string | 合同模板文件名，展示给用户选择 |
| data[].paragraph_count | int | 模板段落总数（可忽略，仅供参考） |
| data[].created_at | string | 模板上传时间 |

### 前端使用

```js
// 页面初始化时调用
const res = await fetch('/api/integration/v1/templates')
const body = await res.json()
if (body.code === 200) {
  // 渲染为下拉框选项
  templates.value = body.data
  // 下拉框显示 name，选中后记录 id
}
```

## 接口 2：执行审查

### 用途

这是核心接口。上传一份待审的 .docx 合同文件，指定比对哪个模板、审查哪些维度，一步完成审查并返回结果。

### 请求

```
POST /api/integration/v1/review
Content-Type: multipart/form-data
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| template_id | int | 是 | 模板 ID，来自"获取模板列表"接口返回的 `data[].id` |
| review_type | int | 是 | 审查维度：`1` 段落比对 / `2` 规则校验 / `3` 完整审查 |
| file | file | 是 | 用户上传的待审合同 .docx 文件，通过 `<input type="file">` 获取 |

**review_type 选择建议：**

| 值 | 含义 | 适用场景 |
|----|------|---------|
| 1 | 仅段落比对 | 只需检查合同正文是否被篡改 |
| 2 | 仅规则校验 | 只需验证填充字段是否符合规范 |
| 3 | 完整审查 | 同时做比对 + 校验，推荐默认使用 |

### 错误码

| code | message | 原因 | 前端处理建议 |
|------|---------|------|-------------|
| 400 | 上传的文件与所选模板不匹配，请确认文件正确 | 合同文件标题与模板不一致 | 提示用户检查文件是否正确 |
| 400 | 只支持 .docx 文件 | 上传的不是 Word 文件 | 限制文件选择器只接受 .docx |
| 400 | review_type 必须为 1、2 或 3 | 传了非法的审查类型值 | 前端限制选项，不要出这个错 |
| 404 | 模板不存在 | template_id 无效或模板已删除 | 刷新模板列表重新选择 |

错误时响应格式统一为：
```json
{"code": 404, "message": "模板不存在", "data": null, "timestamp": 1719312000000}
```

### 前端提交示例

```js
async function submitReview() {
  const formData = new FormData()
  formData.append('template_id', selectedTemplateId.value)
  formData.append('review_type', 3)           // 3 = 完整审查
  formData.append('file', fileInput.files[0]) // 用户选的文件

  const res = await fetch('/api/integration/v1/review', {
    method: 'POST',
    body: formData
    // 注意：不要设置 Content-Type 头，浏览器会自动带 boundary
  })

  const body = await res.json()

  if (body.code === 200) {
    // 审查成功，body.data 结构见下方各 review_type 说明
    renderResult(body.data)
  } else {
    // body.message 即错误描述，直接展示
    showError(body.message)
  }
}
```

## 审查结果说明

### review_type=1 — 段落比对结果

返回 `data.compare`，用于发现合同正文是否被篡改。

#### 响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "document_id": 101,
    "compare": {
      "diffs": [
        {
          "type": "equal",
          "template_range": [0, 14],
          "doc_range": [0, 14],
          "value": "第一条 委托事项"
        },
        {
          "type": "replace",
          "template_range": [15, 21],
          "doc_range": [15, 23],
          "value": "软件开发服务"
        },
        {
          "type": "insert",
          "template_range": [60, 60],
          "doc_range": [55, 78],
          "value": "额外增加的条款内容"
        },
        {
          "type": "delete",
          "template_range": [80, 95],
          "doc_range": [93, 93],
          "value": "被删除的模板条款"
        }
      ],
      "violations": [
        {
          "paragraph": 1,
          "type": "replace",
          "template_text": "______",
          "actual_text": "软件开发服务",
          "template_range": [15, 21],
          "doc_range": [15, 23]
        }
      ],
      "template_text": "模板完整文本...",
      "document_text": "文档完整文本...",
      "paragraph_mapping": {"0": 0, "1": 1, "2": null, "3": 2},
      "inserted_paragraphs": [4],
      "absorbed": {},
      "keyword_matches": []
    }
  },
  "timestamp": 1719312000000
}
```

#### compare 顶层字段

| 字段 | 类型 | 说明 |
|------|------|------|
| diffs | array | 全文逐段的差异片段，按文档位置排序 |
| violations | array | **需要用户关注的违规项**，只包含 replace/insert/delete/关键字命中，不含 equal |
| template_text | string | 模板全文文本，可用于全文对比展示 |
| document_text | string | 上传文档的全文文本 |
| paragraph_mapping | object | 模板段落索引 → 文档段落索引的映射。值为 `null` 表示该段在整个文档中被删除 |
| inserted_paragraphs | int[] | 文档中多出来的段落索引（模板里没有的） |
| absorbed | object | 跨段落吸收映射（填充内容跨多段时使用，通常忽略） |
| keyword_matches | array | 敏感词命中详情，用于在文档中高亮标记 |

#### diffs 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 差异类型，见下表 |
| template_range | int[2] | 该差异片段在模板全文中的 `[起始偏移, 结束偏移]` |
| doc_range | int[2] | 该差异片段在文档全文中的 `[起始偏移, 结束偏移]` |
| value | string | 差异文本内容 |

**diffs 类型说明：**

| type | 含义 | 渲染建议 |
|------|------|---------|
| equal | 内容与模板一致 | 普通文本展示 |
| replace | 模板填充区（如金额、日期）被替换为实际值 | 用对比色高亮，展示"模板原文 → 实际填写的值" |
| insert | 文档中新增了模板没有的内容 | 用新增色标记，需人工确认是否合理 |
| delete | 模板原有的条款被删除 | 用删除色+删除线标记，需人工确认是否允许 |

#### violations 字段

violations 是 diffs 的子集，**只包含需要用户确认的异常项**，等于过滤掉了 type=equal 的条目。新增 `keyword` 类型用于敏感词。

| 字段 | 类型 | 说明 |
|------|------|------|
| paragraph | int | 违规所在的模板段落索引 |
| type | string | 违规类型，见下表 |
| template_text | string | 模板中的原始文本内容。insert 类型时为空 |
| actual_text | string | 文档中的实际文本内容。delete 类型时为空 |
| template_range | int[2] | 模板全文偏移范围 |
| doc_range | int[2] | 文档全文偏移范围 |

**violations type 说明：**

| type | 含义 | 渲染建议 |
|------|------|---------|
| replace | 填充区内容被替换 | 并列显示"模板原文"和"实际填写"，标记为待确认 |
| insert | 文档新增了模板中没有的内容 | 标记为新增，提示用户检查是否合规 |
| delete | 模板原有条款被删除 | 标记为删除，提示用户确认删除是否合理 |
| keyword | 命中敏感词 | 用警告色高亮命中词汇，提示用户修改 |

#### 前端渲染建议

```
违规列表渲染顺序：
  1. keyword  优先级最高 — 用红色/橙色高亮
  2. delete   次高 — 模板条款被删，用删除线
  3. insert   中等 — 有新内容加入，用下划线或绿色
  4. replace  低 — 填充区被替换，通常正常，但仍需确认

每条违规展示三行信息：
  - 违规类型标签（关键字 / 删除 / 新增 / 替换）
  - 模板原文（template_text）
  - 实际内容（actual_text）

配合 template_text（模板全文）和 paragraph_mapping 可实现逐段对比 UI。
```

### review_type=2 — 规则校验结果

返回 `data.validate`，用于校验填充字段是否符合标注规则。

#### 响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "document_id": 101,
    "validate": {
      "results": [
        {
          "paragraph": 1,
          "start_char": 15,
          "end_char": 20,
          "ann_start_char": 10,
          "field_name": "合同金额",
          "actual_value": "50000",
          "rule": "必填+数字+金额(元)↔「大写金额」",
          "pass": true,
          "reason": "",
          "paragraph_text": "甲方应支付50000元整。",
          "template_paragraph_text": "甲方应支付______元整。"
        },
        {
          "paragraph": 4,
          "start_char": 5,
          "end_char": 5,
          "ann_start_char": 3,
          "field_name": "乙方签章",
          "actual_value": "",
          "rule": "必填",
          "pass": false,
          "reason": "必填字段为空",
          "paragraph_text": "乙方（盖章）：",
          "template_paragraph_text": "乙方（盖章）："
        }
      ],
      "document_paragraphs": [
        {"index": 0, "text": "第一条 委托事项"}
      ],
      "template_paragraphs": [
        {"index": 0, "text": "第一条 委托事项"}
      ],
      "paragraph_mapping": {"0": 0},
      "inserted_paragraphs": [],
      "absorbed": {}
    }
  },
  "timestamp": 1719312000000
}
```

#### validate 顶层字段

| 字段 | 说明 |
|------|------|
| results | 各填充字段的校验结论，按 `pass` 排序（未通过的在前） |
| document_paragraphs | 文档全文段落列表 |
| template_paragraphs | 模板全文段落列表 |
| paragraph_mapping | 模板 → 文档段落映射 |
| inserted_paragraphs | 文档中多出的段落 |

#### results 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| paragraph | int | 字段所在的模板段落索引 |
| start_char | int | 文档段落中字段的起始字符位置 |
| end_char | int | 文档段落中字段的结束字符位置 |
| ann_start_char | int | 标注区在模板中的起始字符位置 |
| field_name | string | 字段名称（如"合同金额"），映射标注时的配置 |
| actual_value | string | 从文档中提取到的实际填写的值 |
| rule | string | 生效的校验规则说明 |
| pass | bool | **是否通过校验**。`true` 通过，`false` 未通过 |
| reason | string | 未通过时的原因说明，通过时为空字符串 |
| paragraph_text | string | 文档中该段落的完整文本（上下文） |
| template_paragraph_text | string | 模板中该段落的完整文本（对照使用） |

#### 前端渲染建议

```
结果列表处理：
  1. 先展示未通过（pass=false）的字段 — 用红色标记
  2. 再展示已通过（pass=true）的字段 — 用绿色标记，可折叠

每条字段展示：
  - 字段名称（field_name）
  - 校验规则（rule）
  - 通过/未通过 状态标签
  - 未通过时显示原因（reason）
  - 展示段落上下文（paragraph_text），高亮实际值（actual_value）

reason 为 "" 且 pass=true 表示该项无问题。
reason 非空时直接展示给用户。
```

#### reason 完整列表

| reason | 说明 |
|--------|------|
| 必填字段为空 | 标注为必填的字段没有填写内容 |
| 字数不足：最少 N 字符 | 填写内容少于要求的最少字符数 |
| 字数超限：最多 N 字符 | 填写内容超过允许的最大字符数 |
| 字符类型不符 | 填写内容不符合指定的字符类型（如应为数字却填了中文） |
| 格式不符 | 填写内容不匹配指定的正则表达式 |
| 值不在允许范围内 | 填写值不在预设的允许值列表中 |
| 与「X」不一致 | 两个应保持一致的字段值不同（如合同金额与大写金额） |
| 与「X」不匹配 | 阿拉伯数字金额与中文大写金额不匹配 |
| 「X」未选择任何选项 | checkbox 组中未勾选任何选项 |
| 该条款已勾选，需填写内容 | checkbox 已勾选但关联的填充字段为空 |
| 该条款未勾选，不应填写内容 | checkbox 未勾选但有关联字段被填写了内容 |

### review_type=3 — 完整审查结果

同时包含 paragraph_mapping和 evaluate，结构为两者的并集：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "document_id": 101,
    "compare": { /* 结构同 review_type=1 */ },
    "validate": { /* 结构同 review_type=2 */ }
  },
  "timestamp": 1719312000000
}
```

前端可同时渲染比对差异和校验结论两个区域。

## 完整前端示例

```js
// ============ 第一步：加载模板列表 ============
async function loadTemplates() {
  const res = await fetch('/api/integration/v1/templates')
  const body = await res.json()
  if (body.code !== 200) {
    alert('加载模板失败：' + body.message)
    return
  }
  // body.data 是模板数组，渲染为下拉框
  return body.data
}

// ============ 第二步：提交审查 ============
async function submitReview(templateId, file) {
  const formData = new FormData()
  formData.append('template_id', templateId)
  formData.append('review_type', 3)  // 完整审查
  formData.append('file', file)

  const res = await fetch('/api/integration/v1/review', {
    method: 'POST',
    body: formData
    // 注意：不要手动设置 Content-Type
  })

  const body = await res.json()

  if (body.code !== 200) {
    alert('审查失败：' + body.message)
    return null
  }

  return body.data
}

// ============ 第三步：展示结果 ============
function renderResult(data) {
  // data.compare.diffs     → 逐段差异，按 type 渲染不同颜色
  // data.compare.violations → 违规列表，引导用户逐项确认
  // data.validate.results   → 字段校验结果，按 pass 排序后展示

  // 先展示违规项（compare.violations），再展示字段校验（validate.results）
  for (const v of data.compare.violations) {
    console.log(`段落${v.paragraph}: [${v.type}] ${v.template_text} → ${v.actual_text}`)
  }

  for (const r of data.validate.results) {
    if (!r.pass) {
      console.log(`${r.field_name}: 未通过 — ${r.reason}`)
    }
  }
}
```
